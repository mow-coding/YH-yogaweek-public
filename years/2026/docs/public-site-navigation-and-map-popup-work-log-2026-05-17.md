# 2026-05-17 공개 사이트 헤더 제거 및 시공간타워 팝업 개선 기록

## 목적

공개 사이트 첫 화면의 상단 헤더와 목차 링크가 실제 읽기 흐름에 큰 도움을 주지 않는다는 판단에 따라, 공개 페이지를 더 단순하게 정리했다. 방문자는 별도 목차를 먼저 해석하지 않고, 첫 화면에서 바로 통합 보고서와 핵심 지도로 이동하면 된다.

동시에 `지도 위 시간-공간 큐브`의 점 클릭 경험을 개선했다. 기존 구현은 장소별 타워 하나에 팝업이 묶여 있어, 타워 안의 여러 점을 눌러도 장소 요약 표가 반복해서 보였다. 그러나 타워의 점 하나는 개별 수업 회차를 의미하므로, 점을 눌렀을 때 해당 수업 회차와 연결된 정보를 보여주는 편이 더 자연스럽다.

## 적용한 수정

- 공개 첫 화면의 sticky header와 목차 링크를 제거했다.
- 통합 보고서 HTML 상단의 브랜드/뒤로가기 줄도 제거했다.
- 첫 화면의 hero, 보고서 링크, 핵심 지도 링크, 책임과 담당 섹션은 유지했다.
- 시공간타워의 점을 단순 장식용 `span`이 아니라 클릭 가능한 `button` 요소로 바꿨다.
- 각 점에 `session_id`를 부여하고, JavaScript에서 점별 팝업을 열 수 있도록 했다.
- 점별 팝업에는 수업명, 시간, 장소, 이 회차 예약 수, 리뷰 Hype, 수업명 기준 리뷰 수를 표시한다.
- 공개용 비식별 리뷰 표의 `review_text`를 수업명 기준으로 연결해 팝업 안에서 볼 수 있게 했다.

## 개인정보와 공개 범위

팝업에 넣은 리뷰 본문은 public 산출물인 `obud_reviews_deidentified.csv`에서 가져온다. 이 표에는 실명과 전화번호가 없고, 리뷰 작성자는 공개용 ID로 대체되어 있다.

그래도 지도 HTML 생성 단계에서 전화번호와 이메일 형태의 문자열이 혹시 섞여 들어오면 `PHONE_MASKED`, `EMAIL_MASKED`로 다시 치환하도록 보수적인 방어 처리를 추가했다.

리뷰 연결 기준은 회차가 아니라 `class_base_key`, 즉 표준화된 수업명이다. 같은 수업명이 여러 날짜에 반복 개설된 경우, 리뷰는 특정 날짜 회차가 아니라 그 수업명 전체에 연결된 공개 리뷰로 해석해야 한다.

## 검증 포인트

- 첫 화면과 통합 보고서에 기존 상단 헤더가 남아 있지 않아야 한다.
- 시공간타워의 점을 클릭하면 장소 요약 팝업이 아니라 개별 수업 팝업이 떠야 한다.
- 리뷰가 있는 수업은 팝업 안에 공개용 리뷰 본문이 보여야 한다.
- 리뷰가 없는 수업은 공개 리뷰가 아직 없다는 안내가 보여야 한다.
- public package 품질 게이트에서 금지 계정, 전화번호, API key 패턴이 0건이어야 한다.
