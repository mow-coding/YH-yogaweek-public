# 2026-05-17 공개 웹사이트 가독성 보정 기록

## 목적

공개용 GitHub Pages는 분석 파일을 단순히 올려두는 저장소가 아니라, 외부 이해관계자가 바로 읽는 전시용 페이지다. 따라서 데이터가 맞는 것만으로는 부족하고, 제목, 본문, 표, 안내문이 자연스럽게 읽혀야 한다.

이번 보정은 새로운 분석을 추가한 작업이 아니라, 이미 공개 가능한 상태로 정리한 보고서와 첫 화면을 더 읽기 쉽게 다듬은 작업이다. 특히 한국어 문장이 한 글자 단위로 어색하게 끊기거나, 표 안의 긴 문장이 칸을 밀어내거나, 본문 폭이 넓어 한 줄이 지나치게 길어지는 문제를 줄이는 데 초점을 두었다.

## 확인한 문제

- 첫 화면의 소개 문장이 좁은 폭에서 어색하게 줄바꿈될 수 있었다.
- 통합 보고서 본문 일부가 너무 넓게 펼쳐져 보고서처럼 읽기보다 원문 Markdown을 옮긴 느낌이 남아 있었다.
- 일부 표에서 앞쪽 열을 강제로 한 줄 처리하던 CSS 때문에 긴 설명 열이 답답하게 보일 수 있었다.
- 반대로 모든 표 열을 동일 폭으로 고정하면 책임자/담당자 표의 이메일 같은 짧은 정보가 불필요하게 잘리는 문제가 생겼다.

## 수정 원칙

- 데이터 내용과 해석은 바꾸지 않는다.
- 기존 공개 사이트의 색, 구조, 링크 체계는 유지한다.
- 한국어 문장은 가능한 단어 단위로 줄바꿈되게 한다.
- 표는 칸을 넘치게 하지 않되, 너무 기계적으로 같은 폭을 강제하지 않는다.
- 긴 본문은 한 줄 길이를 줄여 읽는 피로를 낮춘다.
- 모바일 화면에서도 가로 스크롤이 과하게 생기지 않도록 확인한다.

## 적용한 수정

- public 첫 화면과 보고서 HTML에 `word-break: keep-all`을 적용해 한국어 낱말이 한 글자씩 끊기는 현상을 줄였다.
- 첫 화면 제목은 자동 줄바꿈에 맡기지 않고 `연희 요가 위크 / 공개 분석 보고서` 두 줄로 의도적으로 나누었다.
- 본문 문단과 안내문에 `max-width`, `line-height`, `overflow-wrap`을 조정해 한 줄 길이와 행간을 안정화했다.
- 보고서 본문 컨테이너 폭을 줄여 표와 문단이 한 화면에서 더 자연스럽게 읽히도록 했다.
- 표의 첫 번째, 두 번째, 세 번째 열을 무조건 한 줄로 고정하던 규칙을 제거했다.
- 책임자/담당자 표처럼 짧은 조직명과 이름이 들어가는 표에는 해당 열만 한 줄로 유지해 `빅블루 요가` 같은 표기가 어색하게 끊기지 않도록 했다.
- 이해관계자별 메시지 표는 별도 표 유형으로 인식하게 하고, 데스크톱 화면에서 표 전체가 과도하게 넓어지지 않도록 열 폭을 안정화했다.
- 2열, 3열, 4열짜리 설명형 표는 데이터 원표처럼 무조건 넓게 펼치지 않고, 본문 폭 안에서 읽히도록 고정 레이아웃을 적용했다.
- 수업 목록 표는 수업명 열을 넓게, 리뷰 수 같은 숫자 열은 좁게 보이도록 별도 폭 규칙을 두었다.
- 코드 조각과 짧은 식별자는 표 안에서 형태를 유지하되, 일반 설명문은 줄바꿈될 수 있게 조정했다.
- 책임자/담당자 표처럼 짧은 정보가 많은 표에서는 과도한 동일폭 고정을 피했다.

## 검증

- 공개 패키지를 다시 생성했다.
- 공개 품질 게이트를 다시 실행했고 PASS를 확인했다.
- 금지 계정 문자열, 전화번호 패턴, API key 패턴, private/raw 경로 노출 여부를 다시 검사했다.
- 통합 보고서 HTML에서 H1 제목 중복이 없음을 확인했다.
- 브라우저에서 첫 화면과 통합 보고서를 열어 가로 overflow가 없고 주요 표가 화면 안에서 읽히는지 확인했다.

## 결과

이번 작업 이후 공개 사이트는 이전보다 보고서형 웹페이지에 가깝게 정리되었다. 분석 결과 자체를 바꾼 작업은 아니며, 공개 독자가 자료를 읽을 때 줄바꿈과 표 때문에 내용을 오해하거나 중간에 읽기를 포기하지 않도록 만든 가독성 보정 작업이다.
